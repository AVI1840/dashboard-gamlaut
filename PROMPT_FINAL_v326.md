# פרומפט ביצוע סופי - תיקון ושדרוג דשבורד פערי גמלאות v3.26

## רקע
דשבורד React+TypeScript+Vite+Tailwind+shadcn/ui (RTL עברית) לניתוח פערי גמלאות רווחה ברשויות המקומיות בישראל.
נתונים מבוססי ביטוח לאומי, דצמבר 2023-2025.

**כלל ברזל: אל תשבור דפים קיימים. רק הוסף ותקן.**

---

## מצב נוכחי של הדשבורד

### מקורות נתונים קיימים:
- `src/data/welfareData.ts` - דאטהבייס סטטי, 152 רשויות בלבד, 9 סוגי גמלאות
- `public/btl_flat_data.csv` - CSV מגמות, ~285 רשויות, 2,239 שורות
- `src/data/flatData.ts` - parser ל-CSV + פונקציות עזר

### מקורות נתונים חדשים (בתיקיית תיקון_ושדרוג):
- `dashboard_database_v5_FINAL.csv` - 3,058 שורות, 340 ישויות (286 רשויות + 54 מועצות אזוריות), זקנה מתוקנת

- `cbs_regional_councils_full.json` - נתוני למ"ס מלאים על כל מועצה ויישוביה

### דפים קיימים:
- `/` OverviewPage, `/benefit/:benefitId` BenefitAnalysisPage, `/compare` ComparePage

## משימה 1: החלפת מקור הנתונים הראשי (קריטי - ראשון!)

### מה:
החלף את ה-CSV הישן (public/btl_flat_data.csv) בחדש (dashboard_database_v5_FINAL.csv).

### מבנה ה-CSV החדש (20 עמודות, בסדר הזה):
```
0:  Branch                    - סניף ביטוח לאומי (חדש!)
1:  Municipality              - שם ישות
2:  Entity_Type               - "רשות מקומית" / "מועצה אזורית" (חדש!)
3:  Cluster                   - אשכול 1-10 (NaN אם חסר)
4:  Pop_2025                  - אוכלוסייה
5:  Rate_2023                 - שיעור ל-1000 (NaN למועצות)
6:  Rate_2024                 - שיעור ל-1000
7:  Rate_2025                 - שיעור ל-1000 (NaN למועצות)
8:  Change_23_24_Pct          - שינוי %
9:  Change_24_25_Pct          - שינוי %
10: Cumulative_Change_Pct     - שינוי מצטבר %
11: Trend                     - מגמה (עולה/יורדת/יציבה)
12: Z_Score_Cluster           - ציון תקן
13: Small_Pop_Flag            - True/False
14: Benefit_Type              - מפתח (זקנה, סיעוד, נכות...)
15: Benefit                   - שם תצוגה (קצבת זקנה, גמלת סיעוד...)
16: Cluster_Average_Rate_2025 - ממוצע אשכול
17: Gap_from_Cluster_Pct      - פער % מאשכול
18: Operational_Status        - סטטוס (ראה למטה)
```

### ערכי Operational_Status:
```
🔴 גבוה מהאשכול   - Gap > +15%
🔵 נמוך מהאשכול   - Gap < -15%
⚪ תואם אשכול     - -15% <= Gap <= +15%
⚪ ללא סיווג      - חסר אשכול/שיעור
⚪ מדגם קטן       - Small_Pop_Flag = True
```

### פעולות:
1. העתק dashboard_database_v5_FINAL.csv -> public/btl_flat_data.csv
2. עדכן FlatDataRow interface ב-flatData.ts - הוסף Branch ו-Entity_Type
3. עדכן parseCSV() - סדר העמודות השתנה (ראה למעלה)
4. עדכן welfareData.ts:
   - הוסף שדה type ל-Municipality interface: `type?: "city" | "local_council" | "regional_council"`
   - הוסף שדה childSettlements ל-Municipality interface: `childSettlements?: string[]`
   - הרחב municipalities[] מ-152 ל-340 ישויות
   - עדכן benefitData לכל 340 הישויות
   - עדכן nationalStats.totalMunicipalities ל-340

### נתונים קריטיים על מועצות אזוריות:
- למועצות יש רק Rate_2024 (Rate_2023 ו-Rate_2025 ריקים/NaN)
- אין להן Trend, Cumulative_Change, Gap_from_Cluster
- כל 54 המועצות ממופות לסניף (עמודת Branch)
- יש להן Cluster (חלקן) ו-Pop_2025

## משימה 2: תיקון גמלת זקנה (כבר מתוקן בנתונים - רק UI)

### הבעיה:
בנתונים הישנים, זקנה כללה גם שאירים -> שיעור מעל 100% ב-28 רשויות.
ב-CSV החדש, Rate_2025 כבר מתוקן (מקסימום 975, אפס מעל 1000).
אבל Rate_2023 ו-Rate_2024 עדיין מכילים נתונים ישנים (כולל שאירים) - זה תקין כי התיקון חל על 2025.

### פעולות:
1. שנה שם גמלה ב-benefitTypes מ-"קצבת זקנה ושאירים" ל-"קצבת זקנה"
2. עדכן nameEn מ-"Old Age & Survivors" ל-"Old Age Pension"
3. הוסף tooltip בכל מקום שמציג שיעור זקנה: "שיעור מקבלי קצבת זקנה בלבד (ללא שאירים) מתוך אוכלוסיית 65+"
4. ולידציה: ודא שאף רשות לא מציגה שיעור מעל 100% ב-2025
5. עדכן nationalRecipients ו-nationalRatePer1000 בהתאם לנתונים המתוקנים
6. ב-benefitIdToCsvType (BenefitAnalysisPage.tsx): עדכן "old-age": "זקנה" (כבר נכון)

### הערה חשובה על Rate_2023/2024:
בתצוגת מגמות, Rate_2023 ו-Rate_2024 לזקנה עדיין כוללים שאירים.
אפשרויות:
- א. להציג הערה: "נתוני 2023-2024 כוללים שאירים, 2025 זקנה בלבד"
- ב. להסתיר מגמות לזקנה (פחות מומלץ)
- ג. להציג רק Rate_2024 ו-Rate_2025 (מומלץ)

## משימה 3: הוספת 54 מועצות אזוריות (היררכי)

### עיקרון:
מועצות אזוריות הן לא "עוד שורה" - הן מכילות יישובים קטנים.
בטבלאות: מופיעות כשורה רגילה עם תג, לחיצה פותחת רשימת יישובים.

### מבנה נתונים:
```typescript
// עדכון Municipality interface
interface Municipality {
  id: string;
  name: string;
  code: number;
  population: number;
  cluster?: number;
  district: string;
  region: string;
  type?: "city" | "local_council" | "regional_council";
  childSettlements?: string[];  // רק למועצות אזוריות
}
```

### מקור נתוני היישובים:
הקובץ `regional_councils_mapping.json` מכיל לכל מועצה אזורית את רשימת היישובים שלה (מלמ"ס bycode2021).
הקובץ `cbs_regional_councils_full.json` מכיל גם אוכלוסייה לכל יישוב.

### דוגמה למועצה אזורית בנתונים:
```json
{
  "Branch": "טבריה",
  "Municipality": "גולן",
  "Entity_Type": "מועצה אזורית",
  "Cluster": 5,
  "Pop_2025": 20420,
  "Rate_2023": "",       // ריק - אין נתוני 2023
  "Rate_2024": 4.75,     // יש רק 2024
  "Rate_2025": "",       // ריק - אין נתוני 2025
  "Benefit_Type": "נכות"
}
```

### תצוגה בטבלאות:
```
+------------------------------------------+----------+-----------+
| שם רשות                                  | שיעור    | אוכלוסייה |
+------------------------------------------+----------+-----------+
| באר שבע                                  | 5.89%    | 214,661   |
| > גולן  [מועצה אזורית]                  | 0.48%    | 20,420    |
|   |- קצרין (8,380)                       |          |           |
|   |- חספין (2,100)                        |          |           |
|   |- ... 34 יישובים נוספים               |          |           |
| חיפה                                     | 4.82%    | 279,968   |
+------------------------------------------+----------+-----------+
```

### כללי תצוגה למועצות:
1. תג: "מועצה אזורית" (badge) ליד השם
2. הצג רק Rate_2024 (Rate_2023 ו-Rate_2025 ריקים)
3. הצג הערה: "נתונים זמינים: דצמבר 2024 בלבד"
4. אל תציג: מגמה, שינוי מצטבר, פער מאשכול (אין נתונים)
5. לחיצה על שורת מועצה -> accordion/expand עם רשימת יישובים
6. ב-ComparePage: מועצות זמינות לבחירה, עם "(מ.א.)" אחרי השם
7. ב-OverviewPage: מועצות נספרות בסטטיסטיקות הכלליות

## משימה 5: שיפורים נוספים

### 5א. הבהרת תקופות:
הוסף בכל דף תת-כותרת:
```
נתונים: דצמבר 2023 | דצמבר 2024 | דצמבר 2025
```
- ב-OverviewPage: ליד "עדכון: דצמבר 2025"
- ב-BenefitAnalysisPage: בכותרת הגמלה
- ב-TrendPanel: "מגמות דצמבר 2023-2025"

### 5ב. ממשק ניהול נתונים פשוט (אופציונלי):
דף `/admin` עם:
- טבלה editable לעדכון נתונים
- אפשרות לייבא CSV חדש (drag & drop)
- שמירה ל-localStorage
- export ל-CSV/JSON
- כך לא צריך לגעת בקוד בכל עדכון נתונים

### 5ג. פילטרים נוספים (אופציונלי):
- סינון לפי מחוז (district) - כבר קיים בנתונים
- סינון לפי אשכול סוציו-אקונומי (cluster 1-10)
- סינון לפי סוג ישות (רשות מקומית / מועצה אזורית)

### 5ד. הוספת נכות מעבודה ודמי פגיעה:
**הערה: הנתונים האלה לא קיימים ב-CSV החדש!**
ה-CSV מכיל 9 סוגי גמלאות: אבטלה, הבטחת_הכנסה, זקנה, ילד_נכה, ילדים, מזונות, ניידות, נכות, סיעוד.
נכות מעבודה ודמי פגיעה לא כלולים.
**יש להכין נתונים נפרדים לגמלאות אלה לפני שניתן להוסיף אותן.**
